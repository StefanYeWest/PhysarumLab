# Physarum Lab

**Имитационное моделирование самоорганизации _Physarum polycephalum_ при поиске эффективных маршрутов в двумерной среде с препятствиями.**

Интерактивная выставочная web-симуляция: множество простых частиц-агентов перемещается по карте, оставляет виртуальный след, реагирует на след, источники питания и препятствия. За счёт испарения и диффузии следа слабые маршруты исчезают, а часто используемые усиливаются — из простых локальных правил возникает самоорганизующаяся транспортная сеть. Результат сравнивается с классическим алгоритмом A*.

Стек: **React + TypeScript + Vite + HTML Canvas 2D**. Без backend, без базы данных, без внешних API — работает локально в браузере.

---

## 1. Краткое описание

Пользователь работает с картой: размещает стартовую область, источники питания и препятствия (в том числе рисует лабиринт мышью). После запуска симуляции тысячи частиц исследуют среду и постепенно формируют сеть маршрутов между стартом и едой. Исследовательский режим даёт сравнение с A*, метрики эффективности и эксперименты с динамическими препятствиями.

## 2. Скриншот / демонстрация

Скриншоты интерфейса для отчёта складываются в `docs/screenshots/` (см. `docs/testing-plan.md`, как их снять). Базовый сценарий демонстрации описан в разделе 9 ниже.

## 3. Что моделируется

- **Среда** — двумерная сетка клеток (пустые, стены, старт, еда).
- **Агенты** — частицы Physarum с позицией, направлением и скоростью.
- **След** — числовое поле `Float32Array`, оставляемое частицами; испаряется и диффундирует.
- **Пищевой сигнал** — локальное притяжение источников питания.
- **A*** — эталонный кратчайший путь для сравнения.

## 4. Как запустить

Требуется Node.js 18+.

```bash
npm install      # установка зависимостей
npm run dev      # запуск dev-сервера (http://localhost:5173)
npm run build    # production-сборка в dist/
npm run preview  # предпросмотр production-сборки
npm test         # запуск unit-тестов
```

После `npm run build` приложение в `dist/` открывается как статический проект без сервера.

## 5. Как пользоваться

1. Выберите сценарий в панели **Сценарии** и нажмите «Загрузить».
2. Нажмите **▶ Старт** на панели **Управление**.
3. Наблюдайте, как частицы формируют сеть. Меняйте скорость (`0.25x`–`4x`).
4. Инструментами карты рисуйте стены, ставьте старт и еду, инспектируйте клетки.
5. Нажмите **Сравнить с A\*** и **Извлечь маршрут Physarum** на панели **Эксперименты**.
6. Смотрите числа на панели **Метрики**.
7. Нажмите **Добавить препятствие на маршрут** и наблюдайте перестройку сети.
8. Экспортируйте метрики (JSON) и историю (CSV).

## 6. Основные параметры

Настраиваются слайдерами в панели **Параметры модели** (с подсказками):

| Параметр | Смысл | Рекомендуется |
|---|---|---|
| Количество частиц | число агентов | 1500–3000 |
| Скорость частиц | смещение за тик | 1–1.5 |
| Дистанция сенсоров | дальность «чувств» | 5–10 |
| Угол сенсоров | ширина обзора | 30–50° |
| Угол поворота | резкость доворота | 15–25° |
| Количество следа | депозит за тик | 5–10 |
| Испарение следа | скорость исчезновения | 0.01–0.03 |
| Диффузия следа | растекание | 0.1–0.25 |
| Сила притяжения еды | тяга к источнику | 100–250 |
| Радиус притяжения еды | локальность тяги | 15–30 |
| Порог следа для маршрута | граница активной сети | 30–40 |
| Seed | воспроизводимость | любое целое |

Параметры со значком ⟳ (число частиц, seed) пересоздают частицы; остальные применяются «на лету» без сброса.

## 7. Как работает алгоритм

Один тик симуляции:

1. Обновляется пищевое поле от источников.
2. Каждая частица читает три сенсора (вперёд / влево / вправо) и доворачивает к сильнейшему сигналу `trail + food − штраф_за_стену`.
3. Каждая частица пытается сместиться; при столкновении со стеной меняет направление, при долгом застревании — перерождается в старте.
4. После движения частица оставляет след.
5. След испаряется: `trail *= (1 − rate)`.
6. След диффундирует по окрестности 3×3.
7. Обновляются метрики (с троттлингом).

Подробности — в `docs/architecture.md`.

## 8. Сценарии демонстрации

1. **Пустое поле** — базовое соединение старт↔еда.
2. **Простой лабиринт** — поиск пути среди перегородок.
3. **Два источника питания** — ветвящаяся сеть.
4. **Динамическое препятствие** — перестройка маршрута и замер восстановления.
5. **Выставочная демонстрация** — основной демо-сценарий на 2–3 минуты.

## 9. Структура проекта

```text
physarum-lab/
├── public/presets/        # 5 JSON-сценариев
├── src/
│   ├── app/               # App, точка входа, хук useSimulation
│   ├── components/        # React-компоненты UI
│   ├── model/             # 4 класса модели (см. ниже)
│   ├── rendering/         # CanvasRenderer, цвета, отрисовка
│   ├── algorithms/        # A*, математика сетки, ГПСЧ, метрики
│   ├── config/            # конфиг по умолчанию, константы
│   ├── types/             # TypeScript-типы
│   ├── utils/             # clamp, сериализация, скачивание
│   └── tests/             # unit-тесты (Vitest)
└── docs/                  # документация и материалы отчёта
```

**4 основных класса модели** (требование практики):

1. `SimulationEngine` — управление симуляцией, тики, метрики.
2. `WorldGrid` — среда: клетки, след, пищевое поле, испарение/диффузия.
3. `PhysarumParticle` — поведение частицы: сенсоры, движение, след.
4. `PathAnalyzer` — A*, извлечение маршрута Physarum, связность, метрики.

## 10. Тестирование

```bash
npm test
```

Покрыты тестами: A* (пустая карта, препятствия, невозможный путь), `WorldGrid` (проходимость, стены, испарение, диффузия, ограничение, пищевое поле), `PhysarumParticle` (движение, столкновение, респаун, детерминизм), метрики (длина, отклонение, эффективность, связность) и интеграция `SimulationEngine`. Подробно — в `docs/testing-plan.md`.
