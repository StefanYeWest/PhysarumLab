# План и результаты тестирования

## 1. Стратегия

Тестирование состоит из двух частей:

1. **Автоматические unit-тесты** (Vitest) — проверяют алгоритмы и логику модели без UI.
2. **Ручные интеграционные проверки** — проверяют сценарии работы приложения в браузере.

Критерии прохождения: все unit-тесты зелёные, `tsc` без ошибок, в консоли браузера нет runtime-ошибок при основных сценариях, приложение не зависает на стандартных параметрах.

### 2.1.1. Глубокий аудит алгоритма слизевика (`algorithm-audit.test.ts`)

Отдельный набор, целенаправленно проверяющий **корректность самой модели**
Physarum и её сравнение с эталонными решениями.

| Раздел | Что проверяется | Статус |
|---|---|---|
| A. Правило агента | Соответствие канонической модели Jones (2010): впереди сильнее → прямо; бок сильнее → доворот к нему; локальный минимум впереди → случайный поворот; пустое поле → прямо | ✅ |
| A. Движение | Шаг по направлению, депозит только при перемещении, столкновение со стеной (рост `stuckTicks`), перерождение по порогу | ✅ |
| B. Динамика следа | Депозит с ограничением, мультипликативное испарение, обнуление ниже эпсилон, диффузия (не растит массу, не затекает в стены) | ✅ |
| C. Сенсоры | `getSignalAt`: стена/граница → −1000; сигнал = след + поле еды; задокументировано упрощение «точечный сенсор видит сквозь тонкую стену» | ✅ |
| D. Эталон: прямая | Пустое поле: маршрут близок к евклидовой прямой (откл. ~32 % при длине 120) | ✅ |
| D. Эталон: A* | В лабиринтах извлечённый путь **не короче** оптимального A* и формируется (3-wall: A*=237, откл. 26 %; single-gap: A*=175, откл. 22 %) | ✅ |
| D. Классика Physarum | При двух путях между точками сильнее укрепляется **короткий** (масса следа короткого ≈ в 1.5× больше длинного) | ✅ |
| E. Детерминизм | Одинаковый seed → побитово идентичный след | ✅ |
| E. Устойчивость | Прогон 5000 тиков: нет `NaN`, след ограничен `trailMaxValue`, координаты частиц конечны | ✅ |

Ключевой вывод аудита: модель ведёт себя физически корректно — Physarum
никогда не «обыгрывает» A* по длине (его путь всегда ≥ оптимального), при этом
воспроизводит фирменное свойство слизевика — **выбор более короткого из двух
маршрутов** за счёт самоусиления следа.

### 2.2. Баги, найденные и исправленные при сквозном дебаге

| Баг | Симптом | Исправление |
|---|---|---|
| Самоподдерживающаяся петля у старта | Сеть «крутилась» и не доходила до еды | Каноническое правило поворота Jones (случайный поворот при локальном минимуме впереди) |
| Насыщение следа | Низкое испарение → нет контраста, вены не формируются | Испарение поднято до ~0.04; авто-яркость тепловой карты |
| Хрупкое извлечение маршрута | Путь рвался из-за одной клетки; метрика мигала | A* со взвешенной по следу стоимостью + критерий покрытия |
| `step()` не ставил на паузу | Пошаговый режим продолжал считаться запущенным | `step()` принудительно ставит `running = false` |
| Стирание стены оставляло «дыру» | Метки старта/еды под стёртой стеной не восстанавливались | `paintWall(erase)` вызывает `refreshFoodCellTypes()` |
| `resetParams` ломал геометрию | Сброс параметров перетирал размер сетки → рассинхрон мира и конфига | Размер сетки/клетки сохраняется; seed сбрасывается через `setSeed` |
| Нулевое время восстановления | Локальное препятствие не разрывало «широкую» сеть → замер = 0 | При добавлении препятствия след сбрасывается; замер требует перехода «разорвано → восстановлено» |
